## 1. サービス概要

### 1.1 目的

**News Butler** は、複数のRSSフィードを自動収集し、ユーザーが指定したトピックに沿ってAIが記事をまとめ、Webアプリで読めるようにするニュース集約システム。

### 1.2 コンセプト

> **「AI執事があなたの代わりにニュースを読んで、必要な情報だけまとめてくれる」**

ユーザーは「AI執事」を設定するだけで、あとは自動でレポートが届く。

- AI執事ごとに「何を読むか（ソース）」「何にまとめるか（レポート設定）」を決める
- AI執事が定期的にRSSを巡回し、AIでレポートを生成する
- WebアプリでいつでもレポートをMarkdown形式で閲覧できる

### 1.3 MVP（Phase 1）のゴール

> **「RSSを登録して、トピックを指定すれば、AIまとめがWebで読める」**

| やること | 説明 |
|---------|------|
| RSSフィードを登録する | ソース（情報元）をAI執事に紐付けて登録 |
| AI執事を設定する | 名前・説明・分析キーワードを設定 |
| レポート設定を作る | いつ・どんな形式でまとめるかを設定 |
| レポートを読む | AIが自動生成したレポートをWebで閲覧 |

## 2. 機能一覧

### 2.1 AI執事管理

AI執事は「何について知りたいか」を定義する基本単位。内部的には `topics` コレクション。

| 操作 | 説明 |
|------|------|
| 作成 | 名前・説明・アイコン（画像 or カラー）・分析キーワード・言語を設定 |
| 編集 | 名前・説明・アイコン・分析設定を変更 |
| 有効/無効 | 無効にするとRSS収集・レポート生成が停止 |
| 削除 | AI執事と紐付くすべての設定を削除 |

**アイコン設定:**
- 画像アップロード（Firebase Storage に保存、`iconUrl` に URL を格納）
- カラー選択（`iconColor` に hex 値を格納）。画像がない場合はカラー + デフォルトアイコンを表示

**分析設定（`analysisConfig`）:**
- `keywords`: 注目キーワード（AIの要約指示に使用）
- `language`: 要約言語（`ja` / `en`）
- `maxArticlesPerSummary`: 1回の要約で処理する最大記事数（デフォルト: 20）

### 2.2 ソース（情報元）管理

RSSフィードをAI執事に紐付けて管理。

| 操作 | 説明 |
|------|------|
| 登録 | RSS/Atom フィード URL を登録 |
| 編集 | 名前・URL を変更 |
| 有効/無効 | 無効にするとそのソースの収集が停止 |
| 削除 | ソースを削除 |

**自動無効化**: 連続して3回エラーが発生したソースは自動的に無効化される。

### 2.3 レポート設定管理

「いつ・どのようにレポートを生成するか」を定義。内部的には `digest_configs` コレクション。

| 操作 | 説明 |
|------|------|
| 作成 | 設定名・スケジュール・プロンプトテンプレート・対象期間を設定 |
| 編集 | 各パラメータを変更 |
| 削除 | レポート設定を削除 |
| 今すぐ生成 | 手動でレポート生成を即時実行 |

**パラメータ:**
- `schedule`: cron式（例: `0 8 * * *` = 毎朝8時）。`null` の場合は手動のみ
- `promptTemplate`: Vertex AI に渡すプロンプトのテンプレート
- `periodHours`: 何時間分のニュースをまとめるか（デフォルト: 24時間）

### 2.4 RSS収集・AI要約（バックグラウンド）

ユーザー操作なしに自動で実行。

1. Cloud Scheduler が毎時トリガー
2. 全アクティブソースのRSSフィードを取得
3. AI執事（トピック）単位でまとめてVertex AIに送信し要約を生成
4. 要約（`summaries`）をFirestoreに保存

### 2.5 レポート生成

レポート設定のスケジュール、または手動操作で実行。

1. 対象期間内の要約（`summaries`）を収集
2. `promptTemplate` の指示 + 要約内容をVertex AIに送信
3. 生成されたレポート（`articles`）をFirestoreに保存

### 2.6 レポート閲覧

- 全AI執事のレポートを新着順で一覧表示
- レポート本文はMarkdown形式でレンダリング
- AI執事ごとのレポート一覧も確認可能

## 3. UI仕様

### 3.1 用語対応（バックエンド ↔ UI表示）

バックエンドのコレクション名とUI上の表示名は意図的に異なる。

| バックエンド | UI表示 | 説明 |
|------------|--------|------|
| `topics` | AI執事 | RSSを収集・レポート生成するAIエージェント |
| `articles` | レポート | AI執事が生成した最終レポート |
| `digest_configs` | レポート設定 | いつ・どのようにレポートを生成するかの設定 |
| `sources` | ソース（情報元） | RSS フィード |

### 3.2 ルーティング

```
/reports                        # 新着レポート一覧（全AI執事）
/reports/{id}                   # レポート詳細
/butlers                        # AI執事一覧
/butlers/{id}                   # AI執事詳細
/butlers/{id}/sources           # AI執事のソース（情報元）管理
/butlers/{id}/reports           # AI執事のレポート設定一覧
/butlers/{id}/reports/{id}      # レポート設定詳細 + 生成レポート一覧
```

### 3.3 画面構成

| 画面 | URL | 内容 |
|------|-----|------|
| 新着レポート | `/reports` | 全AI執事が生成したレポート一覧 |
| レポート詳細 | `/reports/{id}` | Markdownレンダリング表示 |
| AI執事一覧 | `/butlers` | AI執事カード一覧 |
| AI執事詳細 | `/butlers/{id}` | アイコン（画像/カラー）・名前・説明・ソース一覧・レポート設定一覧 |
| ソース管理 | `/butlers/{id}/sources` | ソースの登録・編集・削除・有効/無効切り替え |
| レポート設定詳細 | `/butlers/{id}/reports/{id}` | 設定の編集・即時生成・生成済みレポート一覧 |

### 3.4 レイアウト

| 画面サイズ | レイアウト |
|-----------|-----------|
| PC（lg以上） | 左固定サイドバー（新着 / AI執事）+ メインコンテンツ |
| モバイル | 下部タブバー（新着 / AI執事）+ 横スワイプで画面切り替え |
